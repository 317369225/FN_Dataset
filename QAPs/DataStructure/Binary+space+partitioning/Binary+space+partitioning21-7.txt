Is it true that writing operating system code and kernel code is extremely difficult? If so, how much do those types of programmers make?
In my experience, it isn't always more difficult.  It depends upon various things, such as the project requirements, and how well that part of the kernel was written before the projects starts.  However, care should be taken not to adversely affect not just the part of the kernel you're working on, but the overall system. The Kernel Programming Guide from the Mac Development Library contains some additional thoughts. Keep Out As far as salaries go, they tend to vary, again, depending upon project requirements, demand for the work in question, etc.
