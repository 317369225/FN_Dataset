Here is some direction A stack gives the reverse operation of the queue. So you have to take out entire list to get the number you would get first using the other. Effectively you might store all numbers some where until you get what you need next. Then you need to put the rest back in order. Try achieving this with a queue with 3 numbers. Do a few runs on paper and it would become evident how you need to store the intermediate numbers and retain their order. Approach is same for queue using stack or stack using queue. 178 Views
