What are the most important data structures in a tech interview?
Hash tables and binary search trees. You should know the running times of the standard operations on each. You should be able to explain how a hash table works and should be able diagram a valid binary search tree on a whiteboard and be able to quickly implement a search operation on a binary search tree. These things shouldn't be hard to you. You should be able to say the phrase "red-black tree" and understand what the purpose of a self-balancing binary tree is but no one will expect you to implement one.
