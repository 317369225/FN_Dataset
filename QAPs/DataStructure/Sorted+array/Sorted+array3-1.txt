If I have a database with 100 billion usernames, how do I efficiently built a sorted array from that to easily perform a binary search?
The data structure that you are looking for is a Patricia Trie with character-level resolution. Well designed, and with front-end transformers (i.e. manipulation of the user names before insertion, search, etc., for the purpose of de-duping) you could hold 100 billion user names in an in-memory data structure using an amount of memory that is a fraction of the size of the database. For example, if you use email addresses as usernames, joe.smith@gmail.com could be “front end translated” to “com/gmail/smith/./joe”.
