Why does JavaScript sort arrays of numbers lexicographically?
This quirk has always felt like a misfeature of the language to me. The only rationale that I can think of is that the array might have elements of mixed types, such as strings and numbers together (or objects or booleans or functions or undefined values, oh my!). In such cases, for better or worse, because JS defines a straightforward way to coerce any value to a string, perhaps it makes sense to prefer lexicographic comparison by default.
