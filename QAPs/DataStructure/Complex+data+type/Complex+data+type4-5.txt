What are the pros and cons of using Amazon Redshift?
Pros •      Columnar and compressed storage •      Probably the fastest database on cloud •      Ability to scale linearly and seamlessly •      Supports almost all standard SQL features •      Less maintenance overheads Cons •      No multi-AZ high availability •      Query concurrency limit of 50 •      Limited node options (only 4 type of nodes) •      Not easy to migrate to another cloud or to on-premise server •      No User Defined Functions, Store Procedures and Triggers •      No indexes and table partitioning •      No enforcement of constraints (Unique, Primary and Foreign keys can be defined. But they are for information purpose only; not enforced)
