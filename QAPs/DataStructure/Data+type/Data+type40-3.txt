When do programmers specify data types?
Well, good question. The issue is that dealing with variables requires all sort of tricks for a program. A strongly typed program is at an advantage because it knows beforehand what it can and cannot do with a variable. Languages without that, like javascript, have to look up each time a variable is used what the type is etc. Why would you use one type over the other? Well, take databases. Imagine you have a million rows. If you would select an integer, it would be 4Meg data. A tinyint would be just 1Meg.
