How do you Explain "Abstract Data Types" in layman terms?
I will give my best shot of being less technical here. So let’s start with an example to set things in context. Assume on a fine day , in some park you decide to stack some pebbles over each other. It is kind of a game for you. You know if you remove something from between, the pile falls off. So the rule is you can remove and add only at the very top. The way you are seeing this game is an “abstraction”. If you were to implement this in a language, you would require multiple pieces. Something that stores what is already in your pile.
