Why do hash tables need to store the key of a record?
Original Question: "Can't we just insert the Value (without the Key), after we calculate the hash for the table (using the Key)?" Yes, you can absolutely do this. This method is called direct addressing. This is a good approach for a small data set. For example to store exam scores of students of a class in a hash table. In a class, roll number is continuous and generally class contains few students. So, we can think of roll number in the range of 1-100.
