What should every programmer know about hash tables and hash functions?
To elaborate on the excellent answer Xanda gave: "Hash table" is a fairly general term, not a one-size-fits-all thing. You can implement one in a number of different ways, optimized to the characteristics of your data set. For example, if inserts are going to be infrequent but lookups frequent, you can pick an implementation that is highly optimized for lookups while making inserts expensive. Or if you know that you will never delete items, you can choose implementations that are appropriate only in that situation.
