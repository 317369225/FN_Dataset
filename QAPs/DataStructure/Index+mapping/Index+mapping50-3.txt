How does a relational DBMS internally store its data?
In general, database should optimize data memory area in one or another way to store most frequently accessed data in memory and to replace idle data with most recently requested data, In case of Oracle, Oracle is using write list and the least recently used (LRU) list. To see mechanism, see below: Memory Architecture Specifically: Organization of the Database Buffer Cache In reality database memory management is much more complex task that just data management. It includes management or redo / logging buffers, SQL shared area and library cache to process SQL statements, private memory areas, etc.
