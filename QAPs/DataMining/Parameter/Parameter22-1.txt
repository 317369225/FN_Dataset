★In Tornado, how can I access POST parameters?self.get_argument(<name>) will get the GET and POST parameters with the name <name>. How can I just get the POST parameters?
If it is absolutely necessary to get only the POST params, you can modify tornado/httpserver.py.  A very simplified example: http://gist.github.com/484774  line 357 self._request.arguments.setdefault("post",[ ]).append({name:value}) The above line generates a dictionary with all POST params without including the GET params and it will include the new dictionary inside the arguments dict.  You can use this like so: class TestHandler(BaseHandler):   def post(self):     self.write(tornado.escape.json_encode(self.request.arguments["post"])) or even: class TestHandler(BaseHandler):   def post(self):     self.write(tornado.escape.json_encode(self.get_argument("post", {}))) Again, this is an oversimplification.  It is more of a start.  In production, it would not be wise to put all the POST params into the self.request.arguments dict (self.get_argument() doesn't play well with dicts, its not elegant, etc).     Embed Quote
